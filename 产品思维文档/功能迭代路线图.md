# DreamAIChat · 功能迭代路线图（v1.0）

> 产品 Owner：邢浩哲  
> 范围：Android 客户端 + 本地后端整体演进

---

## 1. 已完成的核心里程碑

### M1：可运行 Demo（基础对话链路打通）

- 登录/注册流程打通（本地 Spring Boot + JWT）  
- 聊天主界面搭建（消息列表 + 输入框 + 发送按钮）  
- Room 数据库落地（用户 / 会话 / 消息表 + Dao + Repository）  
- Retrofit + OkHttp 接入后端 `/api/chat/send` 接口  

**目标达成**：从“输入文本 → 后端转发 → 返回 AI 文本 → 本地存储”形成闭环。

### M2：体验优化与多模态扩展

- 多图选择 & 压缩 & Base64 上传完整打通  
- 聊天消息支持图片展示（缩略图 + 本地缓存路径）  
- 历史会话列表 + 搜索功能上线  
- 个人中心统计（对话数 / 消息数 / 使用天数）  

**目标达成**：从“简单对话”升级为“可回溯的多轮多模态对话”。

### M3：特色功能集成（模板中心 & 对话图谱）

- 模板中心：四大场景化模板（编程 / 论文 / 医疗咨询 / 翻译）  
  - 通过 `TemplateSelectionFragment` 回填输入框，不直接发送  
- 对话图谱：按时间顺序拼接完整历史对话，生成“总结 + 流程图提示”的长 Prompt  
  - 使用 `pendingPrompt` 回填输入框，交由用户确认发送  

**目标达成**：在不增加前端负担的前提下，把“高级 Prompt 工程能力”包装为易用的 UI 功能。

### M4：工程化与文档体系建设

- 完成架构设计白皮书 / 技术选型最佳实践 / 部署指南 / API 集成说明  
- 故障排查文档沉淀关键 bug（如历史丢失、资源链接、iconGravity 等）  
- AI 指令工程实践报告：系统化记录人机协作与 Prompt 迭代过程  

**目标达成**：让项目具备“可被他人理解和快速上手”的工程化文档能力。

## 2. 下一阶段迭代规划

### P0：线上可用性（后端上云 + 稳定访问）

- 将 Spring Boot 后端部署到云服务器（如火山引擎 ECS）  
- 为训练营评审与同学提供一个稳定的公网入口  
- 在部署指南中补充“云端部署 + 联网诊断”完整流程
- 清理仓库冗余包（backend.zip、gradle-8.13-bin.zip 等），保证交付体积与整洁度

### P1：模型策略与路由面板

- 在 UI 侧重新引入“自动 / 极速 / 质量优先”模型策略选择  
- 与后端 `ModelRouterService` 的 provider 配置联动  
- 展示当前策略与延迟、成功率等基础指标

### P1：会话管理增强

- 历史会话标签 / 收藏 / 归档功能  
- 批量删除、按标签筛选  
- 导出单次对话为 Markdown / PDF，用于学习复盘

### P2：运营 & 数据面板

- 在 Profile 或独立页面展示：  
  - 最近 7 天对话数折线图  
  - 模型使用占比（DeepSeek vs 豆包）  
  - 高频使用的模板类型  
- 为后续产品决策提供数据基础。

## 3. 中长期展望

1. **多端协同**：  
   - 在保持 Android 端稳定的基础上，考虑 Web/H5 版本的轻量实现；  
   - 引导后续团队成员基于统一后端 + 通用 Prompt 体系扩展其他终端。

2. **插件化模板市场**：  
   - 将内置的 4 个模板抽象为“模板插件”；  
   - 后期允许用户自定义模板并保存/分享。

3. **更深层的 Prompt 工程支持**：  
   - 在 UI 中暴露“系统提示/角色设定”等高级选项；  
   - 提供 Prompt 版本管理与回滚能力。

---

本路线图会根据实际开发进度、训练营反馈和真实用户体验持续更新。当前版本重点展示：我们已经完成的工程化基础、特色功能落地，以及对后续演进路径的清晰规划。***
